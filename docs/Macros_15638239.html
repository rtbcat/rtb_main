RTB Cat Help Wiki : Macros

Created by Jen Brannstrom, last modified on Sep 14, 2018

General macros

  1. To buy from Google ADX you must add one of the following macros to your URL:  [DBL_ENCODED_CLICK_REDIRECT] or [UNENCODED_CLICK_REDIRECT] or [ENCODED_CLICK_REDIRECT]
    (This applies to display, in-banner, or VAST/VPAID URL's)
  2. [RANDOM_NUMBER] - cachebuster
  3. [DNT] - do not track
  4. [USER_ID] - returns a unique ID of the visitor
  5. [ORGANIZATION_NAME] - the name of the DSP, ie: RTB.Cat
  6. [AGENCY_NAME] - Agency's account name. (If your Agency has multiple advertiser accounts you can use [ADVERTISER_NAME] to get the specifc ad account)
  7. [ADVERTISER_NAME] - the name of your advertiser account
  8. [CAMPAIGN_NAME]
  9. [STRATEGY_NAME] aka the "ad group"
  10. [AGENCY_ID]
  11. [ADVERTISER_ID]
  12. [CAMPAIGN_ID] - returns a unique campaign ID in our system
  13. [STRATEGY_ID] - a unique id for the strategy
  14. [CREATIVE_NAME]
  15. [CREATIVE_ID] - returns a unique creative ID in our system
  16. [WIDTH] - the creative's width
  17. [HEIGHT] - the creative's height

Optimisation:

  1. [PAGE_URL] - full URL of the impression. WARNING: this can be a lot of data, quickly.
  2. [DOMAIN] - returns the domain name
  3. [PLACEMENT_ID] - returns a unique ad placement ID
    note 1: Google ADX doesn't support placements, all other exchanges do.
    note 2: a placement_id can be any set of number, letters or words - it all depends what the SSP passes us.
  4. [SITE_ID] - returns a unique website ID
  5. [PUBLISHER_ID] - returns a unique ID of the publisher that can contain several websites
  6. [SSP_ID] - returns a unique SSP ID

Auction data:

  1. [REQUEST_ID]
  2. [BID_ID]
  3. [IMPRESSION_ID] - returns a unique impression ID
  4. [CLICK_ID] - returns a unique click ID
  5. [WINNING_PRICE] - returns winning price of the impression
  6. [PROTOCOL] - returns if it was HTTP or HTTPS
  7. [PLACEMENT_ID] - returns a unique ad placement ID - used for AppNexus

Browser:

  1. [USERAGENT]
  2. [IP]
  3. [HTTP_REFERRER] - returns HTTP(S) referrer of the visitor

Mobile specific:

  1. [IFA] - returns a device IFA (Identifier for Advertisers)
  2. [APP_NAME]
  3. [APP_BUNDLE] - returns the Bundle ID
  4. [APP_STOREURL] - returns the store URL of the app

Video:

  1. [VIDEO_DURATION]

Location:

  1. [COUNTRY_CODE] - three letter format iso, uppercase
  2. [COUNTRY] - returns a country name
  3. [REGION]
  4. [CITY]
  5. [PLACE]
  6. [LAT] - Lattitude
  7. [LON] - Longitude
