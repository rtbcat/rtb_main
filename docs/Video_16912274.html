RTB Cat Help Wiki : Video

Created by Jen Brannstrom, last modified on Aug 06, 2017

Types of Video RTB.Cat supports

  • VAST → VAST
  • VAST → in-banner video
  • JS → in-banner video


VAST to VAST buying

To be approved for Google, you must add their click macro [UNENCODED_CLICK_REDIRECT]

eg: http://ssp.lkqd.net/ad?pid=XXX&sid=123456&output=vastvpaid&support=html5flash&execution=any&placement=&playinit=auto&volume=100&width=[WIDTH]&height=[HEIGHT]&dnt=[DO_NOT_TRACK]&ip=[IP]&ua=[USERAGENT]&pageurl=[PAGE_URL]&contentid=[CONTENT_ID]&contenttitle=[CONTENT_TITLE]&contentlength=[VIDEO_DURATION]&contenturl=[CONTENT_URL]&rnd=[RANDOM_NUMBER]&[UNENCODED_CLICK_REDIRECT]



JS or VAST to in-banner video

  1. go to "Creatives" 
  2. select "Display"
  3. select 3PAS (3rd party-ad server)
  4. add name, 
  5. "Advertiser domain": add the domain of the advertiser, eg: volkswagen.com. This is not connected to the ad-server domain, or any tracking domain. It simply declares the domain of the advertiser in the tag.
  6. IMPORTANT: select "in-banner video (auto-play)"
    If you don't declare it, the ad will get blocked.
     

  7. For click tracking, insert one of the following macros. Consult your ad server's documentation for details about which click redirect URL encoding format is required:

    - [UNENCODED_CLICK_REDIRECT]

    - [ENCODED_CLICK_REDIRECT]

    - [DBL-ENCODED_CLICK_REDIRECT]

    For cache busting use:

    - [RANDOM_NUMBER]

    Some SSPs like Google ADX require using macros. Please use [UNENCODED_CLICK_REDIRECT] and [ENCODED_CLICK_REDIRECT] macros to get your creative approved on such SSPs.
