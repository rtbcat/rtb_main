Rules latency

Our platform can use two different sources for statistical data for rules:

  1. AppNexus's reporting service;
  2. RTBCat Impression Tracker service (see RTBCat ImpTracker Service)

Which source to use you can choose in adminitrative console of RTBCat (asc your Admin for support).

Both sources have pros and cons. For example AppNexus reporting service has 3 hours latency so rules based on it will have at least 3 hours latency also.

ATST RTBCat ImpTracker can have up 5% of impressions loss because network issues of users' browsers as it does not act on auction level and can track only actually displayed impressions. Though its latency is about 5 minutes.


Rules latency

RTBCat runs rules every 10 minutes but as they rely on statistical data of a campaign/placement the real latency can be from 5 minutes to 3 hours.

It all depends on the following:

  • AppNexus has latency of up to 15min. (approx.) to apply any changes to campaign targeting.
  • RTBCat has two sources of statistics: AppNexus reporting services and RTBCat Impression/Click/Conversion tracker.
  • If advertiser decides to use the RTBCat trackers - the statistics latency will be about 6 minutes (AppNexus may charge an additional $0.02 CPM charge for adding this)
  • If advertiser decides to use AppNexus' own data and not use RTBCat trackers - the latency will be about 3 hours as this is the latency of AppNexus Reporting Service.


Rules behavior

 

Each rule of the campaign acts independently from other campaigns’ rules and from rules of same campaign.

Rule can only set placement state: BLACKLISTED, WHITELISTED, SUSPENDED.

Rule is not able to clear placement state (i.e. make it has no state).

Only placements in state SUSPENDED can be UNSUSPENDED (i.e. has no state) by timeout set in the rule.

Once rule been applied to a placement it remembers placement’s statistical data within the campaign and next time it will wait for new statistic been collected until this new data will indicate that the placement is eligible to this rule. That means that if some placement will be firstly blacklisted or suspended or whitelisted and then its state will be changed manually or automatically (for suspend case) next time same rule will be applied to this placement only after new statistics will be collected (it does not matter how long it will take). Though other rules will be applied if the placement is eligible.
