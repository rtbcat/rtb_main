Quick Menu Setup

To start integrating this menu, there are 4 necessary files :

  • css/megamenu.css
  • js/jquery.js (or your own copy of jQuery)
  • js/megamenu_plugins.js (already minified)
  • js/megamenu.js (or js/megamenu.min.js - the minified version)

If you choose to support IE6 and IE7, there's an additional file that must be included, megamenu_ie.css

In your HTML file, just make sure that you link properly to each of the above files.

In the folder img, there are a few images that should be included too, just take the ones you need (icons, elements, etc.)

Now it's time to add the HTML : in the package, there are 12 HTML files all showing a possible use / variant of the menu. All of them use exactly the same CSS, most of the HTML markup (except a few classes for the menu variants) and the difference is the small javascript code in the head (enabling the various effects).

We'll distinguish 3 main variants : the fixed menu, the regular menu (both covering the page content) and the "pushing" menu (pushing down the following page content).

The fixed menu is contained in :

<div class="megamenu_wrapper megamenu_fixed">

    <ul class="megamenu">

    MEGA MENU CONTENT

    </ul>

</div>

The regular menu is contained in :

<div class="megamenu_wrapper megamenu_container">

    <ul class="megamenu">

    MEGA MENU CONTENT

    </ul>

</div>

The "pushing" menu is contained in :

<div class="megamenu_wrapper megamenu_container megamenu_push">

    <ul class="megamenu">

    MEGA MENU CONTENT

    </ul>

</div>

Just by changing a few classes you'll be able to switch between 3 versions of the menu. All manue variants require the common class megamenu_wrapper.

Each HTML page has a few lines of script in the head. You can use it to set some options and the comments will help you to modify them.

See the sections below for more informations about the different effects that can be applied to the drop downs.

$(document).ready(function($){
	$('.megamenu').megaMenuReloaded({
        menu_speed_show : 300, // Time (in milliseconds) to show a drop down
        menu_speed_hide : 200, // Time (in milliseconds) to hide a drop down
        menu_speed_delay : 200, // Time (in milliseconds) before showing a drop down
        menu_effect : 'open_close_slide', // Drop down effect, choose between 'hover_fade', 'hover_slide', 'click_fade', 'click_slide', 'open_close_fade', 'open_close_slide'
        menu_easing : 'jswing', // Easing Effect : 'easeInQuad', 'easeInElastic', etc.
        menu_click_outside : 1, // Clicks outside the drop down close it (1 = true, 0 = false)
        menu_show_onload : 0, // Drop down to show on page load (type the number of the drop down, 0 for none)
        menubar_trigger : 1, // Show the menu trigger (button to show / hide the menu bar), only for the fixed version of the menu (1 = show, 0 = hide)
        menubar_hide : 0, // Hides the menu bar on load (1 = hide, 0 = show)
        menu_responsive : 1, // Enables mobile-specific script
        menu_carousel : 0, // Enable / disable carousel
        menu_carousel_groups : 0 // Number of groups of elements in the carousel
	});
	$('#megamenu_form').ajaxForm({target:'#alert'}); 
});

All your content can be organized into columns (up to 12), but the basic structure for each drop down is the same.

Let's start with a couple of containers that will hold our menu :

<div class="megamenu_wrapper megamenu_container">

    <ul class="megamenu">

    MEGA MENU CONTENT

    </ul>

</div>

Inner Tabs

You'll find in all HTML files examples of tabs when you open for example the first drop down. You can see that the content is organised into panels allowing you to add as much content as you need. It requires some additional markup but the columns grid explained above remains the same here. It means that each panel will contain a structure similar to a regular drop down. Let's take back our basic drop down structure :

<div class="megamenu_wrapper megamenu_container">

    <ul class="megamenu">

        <li><span class="drop">Menu Item</span>

            <div class="megamenu_fullwidth">

            DROP DOWN CONTENT

            </div>

        </li>

    </ul>

</div>

And let's add 2 panels as well as the top navigation to switch between each panel :

<div class="megamenu_wrapper megamenu_container">

    <ul class="megamenu">

        <li><span class="drop">Menu Item</span>

            <div class="megamenu_fullwidth">

                <div class="megamenu_tabs">
                
                    <ul class="megamenu_tabs_nav">
                        <li><a href="#section1" class="current">Section 1</a></li>
                        <li><a href="#section2">Section 2</a></li>
                    </ul>
        
                    <div class="clear"></div>

                    <div class="megamenu_tabs_panels">

                        <div id="section1">
                        
                        PANEL 1
                        
                        </div>

                        <div id="section2">
                        
                        PANEL 2
                        
                        </div>
                    
                    </div>
                
                </div>

            </div>

        </li>

    </ul>

</div>

In each section, you can use regular columns as shown above, it works like any other drop down and should fill the whole width every time.

Since the version 2.0 of this menu, it's no longer necessary to set up the tabs in the scripts.

Now it's all ready for your own content, the HTML files show a couple of usages under "PANELS" and "MISCELLANEOUS".

Menu Carousel

As we saw earlier, the basic structure of the menu is an unordered list and each element can contain a drop down. Now there's an additional option allowing you to add as many menu items as you want, in some HTML pages you'll see that the menu bar has arrows on each side. On mouse click, it scrolls to the next set of menu items.

To enable this option, all you have to do is first to add your menu items, and you'll see that they don't fit naturally into the menu bar if you add many of them. Once done, you'll need to enable the carousel in the script placed in the head part of the HTML files :

menu_carousel : 1, // Enable / disable carousel
menu_carousel_groups : 2 // Number of groups of elements in the carousel

The first option requires the value '1' to enable the carousel and the second option is the number of groups (or 'slides') that will contains your menu elements. For example using the value '2' will split the menu elements into 2 groups that you can slide with the left and right arrows.

Menu Scrollbars

You can use custom scrollbars within the drop downs that can help you to keep all the content visible / accessible in the case that a drop down is too tall. For that, let's take a regular drop down :

<div class="megamenu_container">

    <ul class="megamenu">

        <li><span class="drop">Menu Item</span>

            <div class="megamenu_fullwidth">

            DROP DOWN CONTENT

            </div>

        </li>

    </ul>

</div>

And let's add a few containers / classes to enable the scrollbars :

<div class="megamenu_container">

    <ul class="megamenu">

        <li><span class="drop">Menu Item</span>

            <div class="megamenu_fullwidth megamenu_scrollbars">
            
                <div class="megamenu_scroller_container">
                
                    <div class="megamenu_scroll_area">

                    DROP DOWN CONTENT

                    </div>
        
                </li>

            </div>

        </li>

    </ul>

</div>

The javascript will take care of the scrollbars and older browsers will show regular scrollbars. The same happens if javascript is disabled, all the content remains accessible thanks to the regular scrollbars, and if you use the tabs, their content won't be visible without javascript enabled.

CSS Files and Structure

In the css folder, you'll find 2 files :

  • megamenu.css : contains all required (and optional) styling for the mega menu.
  • megamenu_ie.css : this file is for Internet Explorer 6 and 7 only (if you want to support them) and fixes some layout issues.

So the most important CSS file is megamenu.css and it's divided into 7 parts, modifying this file may affect the functionnality of the menu so it's better to make a copy before doing customizations :

  • 00 PAGE STYLING (REMOVABLE) : this code styles the page, it's used for demonstration purposes only and can be safely removed.
  • 01 MENU BAR : It's the code for the menu container and menu items.
  • 02 DROP DOWN CONTAINER : All drop downs use a common class (megamenu_fullwidth) so modifying this class will affect all drop downs, this part is also required to show the drop down on mouse hover (if javascript is disabled).
  • 03 DROP DOWN SCROLLBARS : Scrollbars used within the drop downs.
  • 04 COLUMNS GRID : Here are the containers used to organize your content into columns (up to 12).
  • 05 CONTENT & TYPOGRAPHY : This is the largest part of the CSS, changing or removing styling into this section won't affect the drop down core functionnality, I've put together a lot of typography examples such as lists, images, headings, etc.
  • 06 DROP DOWN TABS : It's the code used to style the tabs within the drop downs.
  • 07 CONTACT FORM : This part styles the contact form and all elements such as feedback messages appearing when using the form.
  • 08 MOBILE DEVICES : All the markup that make the menu mobile-friendly, responsive.

JavaScript

This menu imports four Javascript files.

  1. jquery.js
  2. jquery.easing.js
  3. megamenu_plugins.js
  4. megamenu.js

jquery.js is the necessary framework, there are chances that you're already using it so there's no need to use it twice.

jquery.easing.js is a set of easing effects.

megamenu.js enables the drop down functionnality working on mouse click or mouse hover. Just link to this file after the jQuery framework and set the options situated in the head of the HTML files.

megamenu_plugins.js contains the scripts used for :

  • The Nanoscroller (scrollbars within the drop downs)
  • HoverIntent
  • Hammer.JS (for better mobile compatibility)
  • The contact form

Also keep in mind that the contact form works with sendmail.php

Using this menu doesn't require any modification in the javascript files but in the head part of each HTML file, you'll find a small script. It lets you choose a different effect applied to the menu and some other options. Let's take an example :

$(document).ready(function($){
	$('.megamenu').megaMenuReloaded({
        menu_speed_show : 300, // Time (in milliseconds) to show a drop down
        menu_speed_hide : 200, // Time (in milliseconds) to hide a drop down
        menu_speed_delay : 200, // Time (in milliseconds) before showing a drop down
        menu_effect : 'open_close_slide', // Drop down effect, choose between 'hover_fade', 'hover_slide', 'click_fade', 'click_slide', 'open_close_fade', 'open_close_slide'
        menu_easing : 'jswing', // Easing Effect : 'easeInQuad', 'easeInElastic', etc.
        menu_click_outside : 1, // Clicks outside the drop down close it (1 = true, 0 = false)
        menu_show_onload : 0, // Drop down to show on page load (type the number of the drop down, 0 for none)
        menubar_trigger : 1, // Show the menu trigger (button to show / hide the menu bar), only for the fixed version of the menu (1 = show, 0 = hide)
        menubar_hide : 0, // Hides the menu bar on load (1 = hide, 0 = show)
        menu_responsive : 1, // Enables mobile-specific script
        menu_carousel : 0, // Enable / disable carousel
        menu_carousel_groups : 0 // Number of groups of elements in the carousel
	});
	$('#megamenu_form').ajaxForm({target:'#alert'}); 
});

The menu_effect lets you choose between 6 different options :

  • hover_fade : the drop downs fade in on mouse hover, fade out when the mouse is outside the menu.
  • hover_slide : the drop downs slide down on mouse hover, slide up when the mouse is out.
  • click_fade : the drop downs fade in on mouse click, fade out when the mouse is outside the menu.
  • click_slide : the drop downs slide down on mouse click, slide up when the mouse is out.
  • open_close_fade : the drop downs fade on mouse click and remain visible until the user clicks on the same menu item to close it (fade out).
  • open_close_slide : the drop downs slides down on mouse click and remain visible until the user clicks on the same menu item to close it (slide up).

There are a few other values that you may want to adjust in the code above, let's see them in details :

  • menu_speed_show : duration to show a drop down (in milliseconds).
  • menu_speed_hide : duration to hide a drop down (in milliseconds).
  • menu_speed_delay : delay before showing a drop down (in milliseconds).
  • menu_easing : easing effect to show or hide the drop downs (see the official plugin page for more examples).
  • menu_click_outside : if clicks outside the drop downs close them.
  • menu_show_onload : this option works for the effects 'open_close_fade' and 'open_close_slide', it allows you to show any drop down on page load, for example if you replace 0 by 2, the second drop down will be visible on page load and will remain until the user clicks on the menu.
  • menubar_trigger : shows a button to toggle the menu bar.
  • menubar_hide : hides the menu bar when the page loads if set to "1".
  • menu_responsive : when set to '1', it enables a part of the script necessary for better mobile compatibility.
  • menu_carousel : can be used to add more items in a full menu bar (so it makes the menu 'scrollable').
  • menu_carousel_groups : this option is used along with the previous one to define how many groups of items the menu bar will contain.

Contact Form

Using the contact form requires a single change : you'll need to add your email address. For that, open the file sendmail.php and find the line :

$sendto = abc@xyz.com;

Just put your email address there, it should be at the line 59.

If you want to translate the feedback messages appearing when sending an email, just scroll down this same file and you'll find sentences like "Please enter your name" or "Please enter a valid email address". You can of course change them there.
